Leaderboards

Epic Online Services Interface: Leaderboards Interface

The Leaderboards Interface gives developers using Epic Online Services (EOS) the ability to rank scores from their entire player base, so that players can compete with their friends or other players worldwide for the top score. Each game can support multiple leaderboards, collecting scores from different sources, and ranking them with different scoring modes.

Functions

These functions are provided for handling leaderboards:

  • eos_leaderboards_copy_leaderboard_definition_by_index
  • eos_leaderboards_copy_leaderboard_definition_by_leaderboard_id
  • eos_leaderboards_copy_leaderboard_record_by_index
  • eos_leaderboards_copy_leaderboard_record_by_user_id
  • eos_leaderboards_copy_leaderboard_user_score_by_index
  • eos_leaderboards_copy_leaderboard_user_score_by_user_id
  • eos_leaderboards_get_leaderboard_definition_count
  • eos_leaderboards_get_leaderboard_record_count
  • eos_leaderboards_get_leaderboard_user_score_count
  • eos_leaderboards_query_leaderboard_definitions
  • eos_leaderboards_query_leaderboard_ranks
  • eos_leaderboards_query_leaderboard_user_scores

Constants

These are the constants used for leaderboards:

  • EOS_LEADERBOARD_AGGREGATION
  • EOS_LEADERBOARDS_TIME_UNDEFINED

Structs

These are the structures used by this API:

  • LeaderboardDefinition

  • LeaderboardUserScore

eos_leaderboards_copy_leaderboard_definition_by_index

Epic Online Services Function: EOS_Leaderboards_CopyLeaderboardDefinitionByIndex

This function fetches a leaderboard definition from the cache using an index.

Note

Requires a previous call to eos_leaderboards_query_leaderboard_definitions to store values in cache.


Syntax:

eos_leaderboards_copy_leaderboard_definition_by_index(index)
Argument Type Description
index Real Index of the leaderboard definition to retrieve from the cache




Returns:

LeaderboardDefinition


Example:

var _count = eos_leaderboards_get_leaderboard_definition_count();
for(var i = 0 ; i < _count ; i++)
{
    var _struct = eos_leaderboards_copy_leaderboard_definition_by_index(i);
    var _leaderboard_id = _struct.leaderboard_id;
}
The above code shows an example of how the function should be used. The leaderboard definition data is returned providing an leaderboard index.

Requires a previous call to eos_leaderboards_query_leaderboard_definitions to store values in cache.


Syntax:

eos_leaderboards_get_leaderboard_definition_count()




Returns:

Real


Example:

var _count = eos_leaderboards_get_leaderboard_definition_count();
for(var i = 0 ; i < _count ; i++)
{
    var _struct = eos_leaderboards_copy_leaderboard_definition_by_index(i);

eos_leaderboards_query_leaderboard_definitions

Epic Online Services Function: EOS_Leaderboards_QueryLeaderboardDefinitions

This function queries for a list of existing leaderboards definitions including their attributes. Once the callback has been fired with a successful EOS_RESULT, it is possible to call one of the following functions:

  • eos_leaderboards_copy_leaderboard_definition_by_index
  • eos_leaderboards_copy_leaderboard_definition_by_leaderboard_id
  • eos_leaderboards_get_leaderboard_definition_count

This function operates asynchronously, which means that it does not immediately return the requested result. Instead, upon completion of the task, it will trigger the Social Async Event.


Syntax:

eos_leaderboards_query_leaderboard_definitions(user_id, start_time, end_time)
Argument Type Description
user_id String Product User ID for user who is querying definitions. Must be set when using a client policy that requires a valid logged in user. Not used for Dedicated Server where no user is available.
start_time Int64 An optional POSIX timestamp for the leaderboard's start time, or undefined.
end_time Int64 An optional POSIX timestamp for the leaderboard's end time, or undefined.




Returns:

Real


Triggers:

Social Async Event

Key Type Description
type String "eos_leaderboards_query_leaderboard_definitions"
status EOS_RESULT The status code for the operation. EOS_RESULT.SUCCESS indicates that the operation succeeded; other codes indicate errors
status_message String Text representation of the status code
identifier Real The asynchronous listener ID


Example:

identifier = eos_leaderboards_query_leaderboard_definitions();
The code sample above saves the identifier that can be used inside a Social Async Event.

if (async_load[? "type"] == "eos_leaderboards_query_leaderboard_definitions")
if(async_load[? "identifier"] == identifier)
{
    if (async_load[? "status"] == EOS_RESULT.SUCCESS)
    {
        show_debug_message(async_load[? "type"] + " succeeded!");
    }
    else
    {
        show_debug_message(async_load[? "type"] + " failed: " + async_load[? "status_message"]);
    }
}
The code above matches the response against the correct event type and logs the success of the task.

EOS_LEADERBOARD_AGGREGATION

Epic Online Services Enum: EOS_ELeaderboardAggregation

These constants represent the different leaderboard aggregation types.

These constants are referenced by the following functions:

  • eos_leaderboards_query_leaderboard_user_scores

These constants are referenced by the following structs:

  • LeaderboardDefinition


Member Description
MIN Scores are aggregated by minimum.
MAX Scores are aggregated by maximum.
SUM Scores are aggregated by the sum of the values.
LATEST Scores are aggregated by the last value.
